# 方法论缺口处理与同行评审可辩护性

> 提取自源文档第九章（四个方法论缺口处理）与第十章（文档验证方法）。本章显式处理混合方法路线中的方法论张力，为同行评审提供学术辩护依据。

## 1. 缺口一 Hedges 1999 在古生态学中的应用限制

Hedges 1999 的 log response ratio 在医学和社会科学中被广泛使用（Google Scholar 引用超 4586 次），但在古生态学中应用极少。**核心原因**：古生态学数据的主要结构——多点时间序列叠加——违反了效应量方法的独立性假设。时间序列中相邻观测点存在自相关（AR1 > 0），不满足独立同分布前提，强行使用效应量会低估不确定性。

**应对策略**：将效应量定位为**条件模块**而非主体方法，仅在数据存在配对比较结构时激活（场景一和场景三的特定条件）。对于纯时序叠加（场景二），严格使用原生综合方法，不引入效应量。

## 2. 缺口二 效应量适用性判据

效应量模块的激活判据是**数据结构而非研究者偏好**。判据树如下：

- 数据是否存在"处理组 vs 对照组"的配对比较结构？
  - **是** → 激活效应量模块。进一步判断：
    - 推断值 vs 真值（场景一）→ log response ratio + BCa + RMSEP
    - 事件前 vs 事件后（场景三）→ BCa 差异检验 + 可选效应量（准实验标注）
  - **否** → 仅使用原生综合方法。多点时序叠加（场景二）→ z-score + 合成 + 蒙特卡洛集合

详见 [effect_size.md](effect_size.md) 第 3 节适用边界。

## 3. 缺口三 准实验设计的效应量解释限制

场景三的事件前后比较具有准实验结构（before-after design），但不满足随机对照试验（RCT）的两个核心条件：**随机分配**（事件的发生非研究者控制）和**同期对照**（事件前后无平行对照组）。因此，场景三中计算的 log response ratio 须解释为**关联强度**而非因果效应。

**具体报告要求**：
- 在效应量结果表中显式标注"准实验设计（before-after, 非随机对照）"
- 在讨论中说明残余混杂变量（如气候变化、人口迁移）可能贡献了观测到的变化
- 在结论中使用"与事件相关联的变化"而非"由事件导致的变化"

## 4. 缺口四 REVEALS 模型的 Python 生态空白

REVEALS 模型（Sugita 2007）在 Python 生态中没有成熟实现，现有工具（如 R 的 LRA 包、NEOTOMA 的在线工具）集中在 R 或 Web 平台。此缺口影响场景一中花粉百分比与土地覆被比例的直接对比。三种应对方案：

| 方案 | 操作 | 优势 | 限制 |
|------|------|------|------|
| rpy2 桥接 | 通过 rpy2 调用 R 的 LRA 包执行 REVEALS 转换 | 直接使用经验证的 R 实现 | 需 R 环境安装（当前被阻断） |
| 手动实现 | 按 Sugita 2007 公式用 numpy 实现核心算法 | 纯 Python，无外部依赖 | 须自行验证正确性，RPP 参数须手动维护 |
| z-score 替代 | 不执行 REVEALS 转换，直接对花粉百分比做 z-score 后比较 | 最简方案，无额外依赖 | 丢失花粉-植被非线性关系信息，须在结果中标注限制 |

**当前推荐方案为 z-score 替代**——在 R 环境被阻断时最可行。须在研究报告中明确标注"未执行 REVEALS 转换，花粉百分比与土地覆被比例之间的非线性关系未被校正"。待 R 环境恢复后，可补充 REVEALS 分析作为敏感性检验。

## 7. 方法-数据匹配审计（10.2 节）

逐一检查三个场景的方法是否匹配其数据结构特征：

- **场景一（配对比较）**：使用效应量方法（log response ratio + BCa），匹配"推断值 vs 真值"的配对结构。通过。
- **场景二（时序叠加）**：严格使用原生综合方法（z-score + 合成 + 蒙特卡洛集合），不引入效应量。避免了对时序数据独立性的违反。通过。
- **场景三（准实验）**：使用 Bootstrap BCa 差异检验 + 可选效应量，明确标注准实验限制。效应量定位为关联强度而非因果效应。通过。
